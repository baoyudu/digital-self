# Digital Self — Update

The user wants to quickly update their digital self with recent changes, not do a full interview session.

## Step 1: Load context

1. Read `me.md` for current summary
2. Read `meta/log.md` — note the date of the last session
3. Read `meta/progress.md` for current state
4. Read all files in `inner-world/` (most likely to need updates)
5. Skim other dimension files for potentially outdated information

## Step 2: Read reference guides

Read `references/writing-guide.md` for file update conventions.

## Step 3: Start the update conversation

Keep it light and focused:

> "It's been [time] since we last talked. What's changed? Could be big things (new job, new city, new relationship) or small things (new hobby, shift in mood, something you've been thinking about)."

Then follow up naturally on what they share. This is a shorter conversation — typically 5-15 minutes. Focus on:

- Changes to current life situation
- Shifts in emotional state or focus
- New developments in ongoing threads from previous sessions
- Anything that makes existing records inaccurate

**Check for staleness:** If any dimension files are more than 3 months old (based on `updated` frontmatter), proactively ask:

> "I also noticed we haven't revisited [topic] in a while. Is what I have here still accurate: [brief summary]?"

## Step 4: Write updates

1. Update the relevant dimension files with changes
2. Update `me.md` — especially the "Current State" section
3. Update `meta/progress.md` if any dimension richness changed
4. Append a brief entry to `meta/log.md` (mark it as "update" type, not "interview")

## Step 5: Close

Keep it brief:

> "Got it, all updated. If you want to do a deeper conversation sometime, just use `/digital-self`."
